# פעולות לשיפור ה-AI וצמצום ההקשר (AI Improvement Actions)

## 1. התקדמות התנהגותית בצעדים קטנים (Behavioral progress in small steps)
* **תיאור הפעולה:** פירוק משימות אדריכליות לשלבים נפרדים המאושרים תחת תוכנית.
* **דוגמה אקטואלית:** פירוק יצירת משחק הזיכרון לשלב לוגי דטרמיניסטי בנפרד (Generator), ורק לאחר מכן יצירת ה-UI.
* **תועלת:** דיוק אדריכלי גבוה ומניעת קוד מסורבל, שמירת הקשר צר וממוקד.

## 2. ליטוש הנחיות למודל בעזרת LLM חיצוני (External LLM refining IDE prompts)
* **תיאור הפעולה:** שימוש במודל חיצוני כדי לנסח בקשה חדה, ממוקדת ומדויקת לפני הזנתה לסוכן הפיתוח.
* **דוגמה אקטואלית:** ניסוח הנחיות לפיתוח פריסת מסך ה-Responsive Mobile First שהפרידו בבירור איזה קוד לשמר ואיזה לשנות.
* **תועלת:** מונע "הזיות הבנה" ומייצר ביצוע מדויק על הפעם הראשונה ללא סבבי תיקונים ארוכים.

## 3. מצב מערה (Caveman mode)
* **תיאור הפעולה:** דרישה מהסוכן לפלט אולטרה-תמציתי שאינו כולל הסברים ארוכים והקדמות.
* **דוגמה אקטואלית:** כלל סבבי התיקונים המהירים בממשק ה-WaiterDashboard נעשו תחת בקשה זו.
* **תועלת:** מקטין פליטת טוקנים (Token output), משמר Context פנוי לקוד עצמו, והופך את עבודת הפיקוח של המשתמש למהירה וממוקדת.

## 4. בחירת מודל על פי מורכבות המשימה (Model choice by task complexity)
* **תיאור הפעולה:** התאמה דינמית של בחירת המודל (למשל מודל עמוק ואיטי לעומת מודל מהיר) בהתאם למשימה.
* **דוגמה אקטואלית:** החלפת מודלים בעת יצירת ארכיטקטורה חדשה או עריכת קובץ ענק כמו `WaiterDashboardClient` לעומת עבודת סטיילינג רגילה.
* **תועלת:** איזון נכון בין אמינות, שמירה על קוד קיים בלי מחיקות, וזמן תגובה.

## 5. קריאת קבצים ממוקדת והגבלת טווח עריכה (Targeted file reads and restricted edit scope)
* **תיאור הפעולה:** שימוש בהוראות מפורשות בנוסח "Inspect only" ו-"Edit only".
* **דוגמה אקטואלית:** בעת ריפקטור ה-UI לדשבורד, המודל הורשה לערוך אך ורק את קובץ הקליינט הספציפי, ונאסר עליו לערוך קבצי לוגיקה או מסד נתונים.
* **תועלת:** מניעת "השחתת קוד" (Collateral damage) ושמירה על יציבות של קבצי תשתית.

## 6. קריאת כישורים ספציפיים לפני משימות ממוקדות (Relevant Skill use before tasks)
* **תיאור הפעולה:** אילוץ הסוכן לקרוא SKILL.md ספציפי רגע לפני הפיתוח עצמו.
* **דוגמה אקטואלית:** קריאת ה-Skill `responsive-learning-ui` לפני עריכת בעיות כריתת הטקסט (Truncation) בממשק המשחק הנייד.
* **תועלת:** הטמעת סט חוקי זהב רלוונטיים-בלבד לאותו רגע, מה שמונע העמסה של עשרות חוקים לא רלוונטיים על זיכרון המודל.
