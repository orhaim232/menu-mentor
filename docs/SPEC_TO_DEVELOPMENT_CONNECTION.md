# חיבור מסמך האפיון לתוכנית הפיתוח (Spec to Development Connection)

## 1. מטרת הפרויקט (Main Project Goal)
MenuMentor היא פלטפורמת הדרכה ולמידה מבוססת AI לצוותי אירוח ומלצרים, שמטרתה להפחית את זמן הקליטה, למנוע טעויות זמן-אמת, ולתרגל סיטואציות משירות לקוחות הכוללות אלרגנים והמלצות לתפריט.

## 2. משתמשים עיקריים (Main Users)
* **מלצר (Waiter):** משתמש קצה שמתרגל שאלות צ'אט, מקבל משוב ונעזר בחיפוש תפריט.
* **מנהל (Manager):** מעלה קבצי תפריט ומעדכן את מאגר הנתונים של המסעדה.

## 3. תכונות ליבה מהאפיון, משימות, ו-Skills תומכים (Core Features)

### תכונה 1: סימולציית תרגול AI למלצרים (AI Simulation Practice)
* **מה התכונה (Spec):** המלצר עונה לשאלה של לקוח והמערכת מחזירה ציון מדויק על דיוק עובדתי וטון שירותי.
* **איך הופך למשימות פיתוח:** יצירת טבלאות ב-Supabase (שלב 1), בניית `POST /api/chat` והגדרת ה-UI (שלבים 2,3 ב-tasks.md).
* **Skills תומכים:** 
  * `menu-simulation-feedback` (הערכת תשובת המלצר).
  * `training-scenario-writer` (יצירת השאלה המקדימה למלצר).
* **מצב (MVP / Wait):** שלב MVP. נדרש להקמה הראשונית כדי להוכיח היתכנות.

### תכונה 2: העלאה ופירוק של תפריטים (Menu Upload & Parser)
* **מה התכונה (Spec):** מנהל מעלה קובץ תמונת תפריט, והמערכת מחלצת מנות, אלרגנים ומרכיבים באופן אוטומטי למסד נתונים סמנטי.
* **איך הופך למשימות פיתוח:** יצירת עמוד Dashboard, טופס העלאה שמפעיל מודל Vision ומזין נתונים למסד (שלב 4).
* **Skills תומכים:**
  * `menu-upload-parser` (חילוץ בטוח ללא "המצאת" אלרגנים).
* **מצב (MVP / Wait):** תכונת Production (יכול לחכות אחרי ה-MVP).

### תכונה 3: חיפוש תפריט מהיר (Fast Semantic Menu Search)
* **מה התכונה (Spec):** מלצר יכול לשאול "מה יש ללא גלוטן וללא תבלינים?" ולקבל תוצאות מדויקות על סמך משמעות הֶקְשֵׁרִית (Semantic similarity) בפחות מ-500ms.
* **איך הופך למשימות פיתוח:** הפעלת `pgvector` ב-Supabase ובניית `POST /api/search` עם UI רלוונטי (שלב 4).
* **Skills תומכים:**
  * `semantic-menu-searcher`.
* **מצב (MVP / Wait):** תכונת Production.
